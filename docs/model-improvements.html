Model Report · Dog Vision API

Rebuilt the breed classifier from a single small model into a calibrated two-model ensemble.

The old classifier was a single MobileNetV2 with a training pipeline that had silent bugs. We fixed the pipeline, moved to two stronger backbones, and combined them. On the same 1,280-image validation set, top-1 accuracy rose from 88.05% to 92.42% — roughly a third fewer wrong guesses.

What we changed, and why it moved the number

Five changes, from the least visible to the most. The pipeline fix came first — without it, the bigger models would have learned from corrupted inputs.

01

Fixed a silent input-scaling bug in the training pipeline

Images were being brightened and shifted after being scaled to 0–1, which turned augmented pictures into near-white noise. Every model trained on that data was learning from garbage. We corrected the order so augmentation runs on real pixels.

02

Replaced the backbone with two modern ones

MobileNetV2 (2.4M parameters) → EfficientNetV2S and ConvNeXtTiny. Larger, more recent vision models that separate visually similar breeds far better.

03

Combined the two models into an ensemble

The API averages both models' predictions. Where one hesitates, the other often decides — which is why the ensemble (92.42%) beats either model alone (89.92% / 90.78%).

04

Calibrated the confidence scores

Temperature scaling (T=0.67 and T=0.73) makes each model's reported confidence match how often it is actually right — so the "unknown breed" threshold and the top-3 percentages mean what they say.

05

Made the evaluation trustworthy

The old figure came from just 25 hand-picked images. Every number here is measured on a held-out validation set of 4,116 images (1,280 sampled for this head-to-head), so the comparison is fair and repeatable.
